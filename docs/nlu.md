# Нейросетевое понимание описания (этап 1)

Слой добавляет к детерминированному `extract.js` нейросетевое понимание свободной
речи, не ломая существующий конвейер. Цель — чтобы новые формулировки и признаки
добавлялись как **примеры фраз**, а не как регекспы.

Всё работает **локально и офлайн** — жёсткое правило проекта. Сети нет.

## Как устроено

```
описание (текст/Vosk)
   │
   ├─ extract.js  — регекспы, быстрый и точный путь (как раньше)
   │                 выставляет уверенные признаки
   │
   └─ nlu.py      — fallback: матчинг ПО СМЫСЛУ к примерам фраз из traits.js
                     дополняет только то, что регекспы не распознали
   ↓
{ values, params, marks, evidence }   ← единый контракт, не изменился
```

Правило слияния: **явное правило побеждает**. NLU не перетирает то, что уверенно
выставил регексп, — объяснимость для оператора сохраняется. Каждый признак от NLU
попадает в `evidence` с фрагментом-источником и уровнем уверенности, как и у
регекспов (панель «Распознанные признаки» работает без изменений).

## Где что

- `js/traits.js` — поле `NLU_PHRASES` (2–4 примера на вариант) и `nluSchema()`.
  Единый источник правды. **Расширение системы = дописать фразу сюда.**
- `nlu.py` — локальный матчер. Бэкенд эмбеддингов выбирается автоматически:
  1. `sentence-transformers` — семантическая модель (синонимы, перефразировки);
  2. `hashing` — бесзависимостный fallback (нужен только numpy), без модели.
- `app.py` — мост `window.pywebview.api.interpret(text, schema)`.
- `js/app.js` — `enrichWithNLU()` вызывает мост после регекспов.
- `tests/nlu_test.py` — офлайн-проверка конвейера и точности.

Доступно в нативном приложении (`.exe` / `Запустить.bat`), как и голос. В чистом
браузере слой отсутствует, сценарий продолжает работать на регекспах.

## Включить семантическую модель (офлайн)

Без модели слой уже работает на hashing-бэкенде. Для лучшего понимания синонимов:

```
py -m pip install sentence-transformers numpy
```

Модель скачивается один раз и далее работает офлайн. Для автономного `.exe`
положите её каталог в `models/nlu` (тогда сеть не нужна вовсе) — `app.py` сам
подхватит путь через `NLU_MODEL_DIR`. Рекомендуемая модель:
`sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`.

## Тест

```
# схема из traits.js
node -e "const fs=require('fs');const w={};eval(fs.readFileSync('js/traits.js','utf8'));process.stdout.write(JSON.stringify(w.FC.traits.nluSchema()))" > schema.json
python3 tests/nlu_test.py schema.json
```

На бесзависимостном hashing-бэкенде базовая точность по тест-набору ~95%;
с семантической моделью выше на перефразировках и падежах.
