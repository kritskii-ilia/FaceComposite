# Фоторобот — конструктор (MVP)

Десктоп-ориентированное **локальное** приложение-конструктор для составления
ориентировочного фоторобота человека по словесному описанию. Оператор работает
со свидетелем/потерпевшим, вводит описание внешности — система собирает стартовый
портрет из параметрических черт, который дальше уточняется вручную.

Это **MVP первого этапа**: не генеративная нейросеть, а управляемый, объяснимый
конструктор (см. бриф в [docs/original-user-prompt.md](docs/original-user-prompt.md)).

## Установка из исходников

```bash
git clone https://github.com/kritskii-ilia/FaceComposite.git
cd FaceComposite
py -m pip install pywebview vosk sounddevice
```

Затем скачать русскую языковую модель Vosk (~45 МБ):

```bash
# В папку models/ проекта
curl -L https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip -o vosk-ru.zip
unzip vosk-ru.zip -d models/
```

Запустить: `Запустить.bat` или `py app.py`.  
Собрать автономный `.exe`: `build_exe.bat` (нужен `pyinstaller`).

---

## Как запустить

Интернет не нужен ни в одном из вариантов; данные хранятся только локально.

**1. Автономное приложение (рекомендуется для выставки/чужих машин).**
Запустить `dist\FaceComposite\FaceComposite.exe` — настоящее десктоп-окно, Python
и установка не требуются. Всю папку `dist\FaceComposite\` можно скопировать на любой
Windows-ПК (нужен лишь системный компонент WebView2, он есть в Windows 10/11 по умолчанию).

**2. Нативное окно через Python (для разработки).**
Двойной клик по `Запустить.bat`. Требуется один раз: `py -m pip install pywebview`.
Пересобрать `.exe` — `build_exe.bat` (нужен ещё `pyinstaller`).

**3. Просто в браузере (самый надёжный fallback).**
Открыть `index.html` двойным щелчком в Microsoft Edge / любом Chromium-браузере.

## Что умеет MVP

- **Карточка дела**: название, оператор, тип субъекта (разыскиваемый/пропавший/иное).
- **Словесное описание** → кнопка «Собрать по описанию» запускает детерминированное
  извлечение признаков из русского текста и собирает стартовый портрет.
- **Демо-пресеты**: чипы с готовыми описаниями (мужчина с бородой, молодая женщина,
  пожилой, девушка, ребёнок, подозреваемый) — портрет собирается в один клик.
- **Голосовой ввод (офлайн)**: кнопка «🎤 Диктовать» — распознавание русской речи
  локально через Vosk, без интернета. Доступно в нативном приложении (`.exe` /
  `Запустить.bat`); в чистом браузере отключено. Распознанный текст сразу идёт
  в извлечение признаков и сборку портрета.
- **Панель «Распознанные признаки»** показывает, *что*, *по какому слову* и
  *насколько уверенно* выставлено — каждый признак помечен «уверенно» (явно назван)
  или «вероятно» (оценка: возраст, форма очков по слову «очки», приблизительная
  величина). Объяснимость — основа доверия оператора.
- **Отмена/повтор** правок (кнопки и Ctrl+Z / Ctrl+Y) — история до 60 шагов.
- **Ручная корректировка**: выпадающие списки по всем основным чертам
  (пол, возраст, тон кожи, форма лица, подбородок, уши, глаза + цвет, брови, нос,
  губы, причёска + цвет, растительность, очки).
- **Тонкая настройка ползунками**: ширина/длина лица, ширина челюсти, высота лба,
  размер и постановка глаз, размер/длина носа, размер/высота рта, высота бровей.
- **Особые приметы**: шрамы, родинки, веснушки, морщины.
- **Подбор «выбери похожее»**: кнопка показывает популяцию вариаций целого лица;
  оператор/свидетель отмечает наиболее похожие, система сближается к цели (whole-face
  refinement в духе EvoFIT/EFIT-V). Закрывает пункт этапа 2 про несколько вариантов
  и итеративное уточнение.
- **Подбор лиц по описанию**: кнопка «Подобрать лица по описанию» собирает галерею
  кандидатов, *согласованных с текстом*: признаки, прямо названные в описании
  (форма лица, нос, брови…), фиксируются **замком** и не меняются, а варьируется
  только то, что свидетель не упомянул — трудновербализуемая геометрия. Любой замок
  можно снять в один клик, чтобы признак тоже начал варьироваться. Выбор похожих →
  сближение → «Применить» → доводка обычными чертами. Это путь «от описания к лицу
  через выборку», а не подгонка единственного портрета.
- **Версии портрета**: снимок текущего состояния; к любой версии можно вернуться.
- **Проекты**: сохранение/список/открытие/удаление локально (localStorage),
  плюс экспорт/импорт карточки файлом `.fcase.json`.
- **Лист-ориентировка**: кнопка «Ориентировка» собирает печатный A4-лист (шапка,
  портрет в рамке, данные дела, перечень признаков, словесное описание, дисклеймер)
  и сохраняет его в PNG — вид официального документа, а не просто картинки.
- **Экспорт результата**: PNG (портрет), SVG (векторно), JSON-проект.

## Почему так сделано (ключевые решения)

- **Стек — локальное web-приложение без сборки.** Для демонстрации это самый
  надёжный и визуально выигрышный вариант (ничего не «отваливается» на чужой
  машине, не нужен Visual Studio), а для будущего продукта — не тупик: тот же
  фронтенд оборачивается в нативное окно Windows (WebView2/Tauri). Подробнее —
  [docs/architecture.md](docs/architecture.md).
- **Рендер — параметрический, а не свопом картинок.** Каждая черта рисуется кодом
  из опорных точек геометрии головы. Это даёт *плавную* ручную правку ползунками
  и гарантированно единый стиль; не нужен внешний арт-пак и нет лицензионных рисков.
  Поверх — мягкий свет/тени (объём), различие геометрии по полу и возрасту.
- **Извлечение признаков — детерминированное (правила), а не нейросеть.** На первом
  операционном этапе важнее объяснимость и контроль оператора.

## Структура

```
icon.html         — исходник иконки (SVG); app.ico — собранная иконка приложения
index.html        — разметка экрана
styles.css        — операторская тёмная тема
js/traits.js      — таксономия признаков (единый источник правды) + модель профиля
js/render.js      — параметрический движок сборки лица (SVG)
js/extract.js     — детерминированное извлечение признаков из русского текста
js/store.js       — локальное хранение, импорт/экспорт, PNG/SVG
js/ui.js          — построение панелей из таксономии
js/app.js         — состояние карточки, цикл перерисовки, обработчики
docs/             — бриф и архитектурные заметки
```

Добавить новый признак = правка `traits.js` (описание) + генератор в `render.js`.
UI и сохранение подхватят его автоматически.

## Дорожная карта (этап 2) — без переписывания

Архитектура заложена так, чтобы наращивать, а не переделывать:

1. **Голосовой ввод (офлайн STT)** — ✅ есть (Vosk, русская модель, `voice.py` +
   мост pywebview). Дальше: потоковый показ распознавания, выбор устройства, пунктуация.
2. **Умная интерпретация** свободной речи: словарь правил расширен (тон кожи,
   падежи и порядок слов в возрасте, отрицания «без бороды/очков», синонимы);
   корректность зафиксирована тестами — [tests/extract-tests.html](tests/extract-tests.html)
   (открыть в браузере: показывает PASS/FAIL по набору фраз). ✅ Есть **слой
   уверенности**: каждый распознанный признак помечен «уверенно» (явно назван)
   или «вероятно» (оценка — возраст, форма очков по слову «очки», приблизительная
   величина) с цветным индикатором в панели «Распознанные признаки». Дальше:
   обучаемые веса вместо эвристики, выделение источника прямо в тексте.
3. **Несколько стартовых вариантов** и **итеративный отбор** (выбор похожих) —
   ✅ есть (`evolve.js`, кнопки «Подбор «выбери похожее»» и «Подобрать лица по
   описанию»). ✅ Есть и **«замки» на признаки** — названные в описании черты
   автоматически фиксируются, остальное варьируется; замок снимается кликом.
   Дальше: сохранение истории поколений и наглядное сравнение ветвей.
4. **Нативная Windows-обёртка** — ✅ уже есть (pywebview + WebView2, `app.py`,
   автономный `.exe`). Дальше: хранение проектов в `%APPDATA%` вместо localStorage
   (контракт `store.js` сохраняется), иконка и инсталлятор.

## Ограничения

- Стиль изображения — **иллюстративный** (открытых фотореалистичных наборов частей
  лица офлайн не существует; фотореализм — отдельная задача этапа 2).
- Инструмент помощи оператору для *ориентировочного* портрета, **не** средство
  автоматической идентификации личности.
